Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

Bezier curves take in a set of an unlimited number of ordered points (n) and returns a smooth curve using a recursive subdivision algorithm. To create the curve, linear interpolation is conducted across each set of consecutive points, creating a new set of n-1 interpolated points. This means for some value t in the range [0,1], each interpolated point is a distance proportional to t between two neighboring points. This process is repeated recursively until just one point remains. That point lies on the Bezier curve. By varying t from 0 to 1, the full path of the curve is traced.

Here is an example 2x2 gridlike structure using an HTML table. Each tr is a row and each td is a column in that row. You might find this useful for framing and showing your result images in an organized fashion.

Part 2: Bezier surfaces with separable 1D de Casteljau

To create a Bezier surface from an nxn grid of control points, the same idea used for Bezier curves is applied in two directions. First, for each of the n rows of 3D control points, a Bezier curve is evaluated using the parameter u. For any specific value of u, this produces n intermediate points, one from each row. These points are then treated as control points for another Bezier curve in the perpendicular direction, evaluated using a second parameter v. By varying both u and v between 0 and 1, points across the entire Bezier surface are generated.

Part 3: Area-weighted vertex normals

Area-weighted vertex normals were calculated in order to implement Phong shading, improving smoothing as shown below. By iterating through the half-edges around a given vertex and calculating the area-weighted normal of each corresponding non-boundary face, the normalized sum of area-weighted vertex normals was calculated.

Part 4: Edge flip

An edge flip is a remeshing operation that takes constant time, changing the orientation of an edge between two triangles. Each triangle has a third vertex that isn't part of the edge. The edge flip transforms the edge to be connecting those two third vertices. My debugging journey was quite short; using a comprehensive check aside a drawing of the before and after states helped with understanding.

Part 5: Edge split

An edge split is a remeshing operation that takes constant time, adding a vertex and resulting in splitting of each of the two triangles on the edge. The edge split creates a vertex at the midpoint of the edge, then connects each triangle's third vertex to the newly created vertex.

Part 6: Loop subdivision for mesh upsampling

Mesh upsampling is used to increase fineness in a mesh, whether for more accurate simulations or a more smooth image. Loop subdivision was implemented as a method for mesh upsampling, using edge flipping and splitting to subdivide each triangular mesh face to four smaller, triangular mesh faces.
Using the provided framework, I determined the old and new vertex positions before adding new edges, performed upsampling using edge splitting and flipping, then committing the vertex positions using the pre-determined values. This implementation is simpler and cheaper, as traversing through a coarser mesh requires fewer objects and operations.
Loop subdivision is a smoothing operation, so any sharp corners are quickly erased and rounded. However, as shown with the cube on the left, while the cube is symmetrical at first, the orientation of the mesh results in an assymetric upsampled cube. While exploring the provided .dae files, I found the cube to be the easiest example to use to discover the loop subdivision behavior. Pre-splitting each edge on the square face results in the faces around that edge to be smoothed less. This results in a pillowed cube. Interestingly, using edge flipping to convert opposite square face edges to have opposite orientation results in a chair-like cyclohexane structure seen in chemistry.
